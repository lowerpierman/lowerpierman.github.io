网络编程时的同步、异步、阻塞、非阻塞？

  • 同步：函数调用在没得到结果之前，没有调用结果，不返回任何结果。

  • 异步：函数调用在没得到结果之前，没有调用结果，返回状态信息。

  • 阻塞：函数调用在没得到结果之前，当前线程挂起。得到结果后才返回。

  • 非阻塞：函数调用在没得到结果之前，当前线程不会挂起，立即返回结果。

30.http和https

http是短连接，服务器响应请求后立即断开

https是http的加密处理(SSL)

首先客户端对http请求到的证书进行验证，使用RSA公钥；

接着根据证书有效期生成随机数，对该随机数使用RSA公钥加密；

在消息体生成后使用消息摘要生成算法如MD5 进行加密，得到RSA签名

将RSA签名发给服务端，服务端解密，再用AES加密，作为密钥；

33.索引

索引的增删：

Create：create table index_name(col primery key(,…) ‘unique’) or create table tab_name(col ‘unique’)
或 alter table tab_name add index_name(col)
Drop：drop index index_name on table_name;

索引的数据结构是B+树，是一种加快查找速率的方式。

为什么用B+树，不使用红黑树，二叉树，B树，Hash查找：一种多叉平衡树，因为二叉树、红黑树等结构存储数据仍会导致树的高度过高，查找效率不如B+树。

索引为什么用B+树不用hash和B？

1.B树在内部节点存储数据，不稳定，存储的索引更少

2.hash适合精度更高的查找，有hash冲突

3.B+树除了多叉外，其叶子节点从小到大由指针连接，如同一个单向链表，非叶子节点不存储数据。

聚簇索引和非聚簇索引区别：数据和索引是否分开存储

  • 聚簇索引：将数据和索引放到一起存储，索引结构的叶子节点保留了数据行。

  • 非聚簇索引：将数据进和索引分开存储，索引叶子节点存储的是指向数据行的地址。

    最左匹配原则 索引(a,b,c) 必须从左到右连续匹配条件，遇到！，=，<,>,between，or，运算符，数据转换，使用函数都会导致索引失效
34.数据库隔离级别及ACID

A：原子 事务中操作的成功与否必须保持一致

C：一致 事务之间的状态必须一致，不可丢失

I：隔离 事务之间并行不能导致操作丢失

D：持久 事务一旦提交操作，则不可能丢失

不考虑隔离性可能出现：1.读脏数据 () 2.不可重复读(锁行) 3.幻读(锁表)

采取四种隔离级别：
读未提交、读已提交、可重复读、串行化

读未提交：啥都没解决；读已提交：解决脏读；可重复读：解决不可重复读；串行化：解决幻读
